# 股票交易界面 Demo｜产品设计说明

## 1. 项目概述

本 Demo 是一个面向 A 股与美股个人投资者的股票交易界面原型，包含 Web 端与 Mobile 端两个独立 HTML 页面。

产品覆盖行情查看、K 线图、盘口、下单面板、持仓、订单记录和风险控制等基础交易功能，并在传统交易流程之外设计了一个差异化复盘功能：「持仓平行宇宙」。

本 Demo 的核心目标不是简单复刻交易页面，而是探索交易产品中一个常被忽视的方向：

> 交易平台不应只帮助用户完成买卖，也应帮助用户理解自己的交易决策。

## 2. 交付文件

- `stock-trading-web.html`：Web 端交易工作台
- `stock-trading-mobile.html`：Mobile 端交易界面

两个页面均为独立 HTML 文件，可直接在浏览器中打开查看。

## 3. 产品定位

本产品定位为：

> 面向 A 股与美股个人投资者的交易与复盘一体化工具。

传统股票交易产品通常重点解决三个问题：行情查看、交易执行、资产管理。但在真实交易过程中，用户的核心痛点并不只发生在“买入”和“卖出”当下，也大量发生在交易前后的心理决策中。

典型场景包括：

- 曾经看好一只股票，但没有真正买入
- 买入后提前卖出，后续股价继续上涨
- 卖出后股价下跌，事后发现自己规避了风险
- 用户反复复盘“如果当时……”，但缺少工具帮助其量化结果

因此，本 Demo 尝试把用户常见的后悔情绪，转化为可量化、可解释、可复盘的交易决策反馈。

## 4. 目标用户

核心用户是有一定股票交易经验的个人投资者，尤其是：

- 经常关注 A 股与美股行情
- 有过“卖飞”“踏空”“追高”“止盈过早”等经历
- 会在交易后复盘自己的操作
- 对交易平台的专业性有要求，同时也愿意接受轻量、有趣的产品表达

这类用户不只关心“现在怎么买卖”，也会持续思考“当时为什么这样决策”以及“下次如何做得更好”。

## 5. 基础交易能力

为了保证 Demo 具备真实交易平台的基础可信度，页面中设计了以下核心模块：

- 股票切换：支持 A 股与美股案例
- 行情展示：展示价格、涨跌幅、资金、事件、操作建议
- K 线图：包含蜡烛图、成交量、均线、最新价标记
- 下单面板：支持买入、卖出、限价单、市价单、条件单
- 快捷仓位：支持 1/4、1/2、3/4、All
- 盘口信息：展示买卖盘深度
- 账户信息：展示总资产、可用资金、持仓市值、盈亏
- 订单记录：模拟成交与挂单状态
- 风险提示：展示仓位、止损、策略、提醒

这些基础能力用于建立交易产品的专业感和完整度，让创新功能建立在真实交易场景之上。

## 6. 核心创新功能：持仓平行宇宙

### 6.1 功能定义

「持仓平行宇宙」是一个交易复盘功能。

用户可以选择一只自己曾经关注或交易过的股票，输入当时的关键时间点，系统会根据历史价格和当前价格，计算另一种决策路径下的结果。

它解决的不是“现在该不该买”，而是：

> 如果当时做了另一个决策，现在会怎样？

### 6.2 入口设计

该功能在产品中被设计为独立入口，而不是直接混入下单流程。

- Web 端：底部 API Bar 展开
- Mobile 端：底部 Sheet 展开

这样设计有两个原因：

- 不干扰用户正常看盘和交易操作
- 让「持仓平行宇宙」成为交易后的独立复盘工具

它既是交易产品的一部分，又有清晰的功能边界。

## 7. 两类复盘场景

### 7.1 想买没买

用户选择一只曾经想买但最终没有买入的股票，输入股票、想买日期、计划投入金额和当时价格。

系统计算：

- 如果当时买入并持有到现在，当前价值是多少
- 相比没有买入，本可以多赚多少钱
- 或者幸好没有买入，避免亏损多少钱

结果分为两类：

- 股价上涨：`😭我还没上车呢`
- 股价下跌：`😍躲过一劫`

该场景对应的是用户的“踏空”和“未执行”问题。

### 7.2 卖早了

用户选择一只曾经买入并卖出的股票，输入股票、买入日期、卖出日期、投入金额、买入价格和卖出价格。

系统计算：

- 如果当时没有卖出，持有到现在会怎样
- 如果卖出后继续上涨，用户少赚了多少钱
- 如果卖出后下跌，用户反而多赚了多少钱

结果分为两类：

- 卖出后继续上涨：`😭我卖飞了`
- 卖出后下跌：`😍还好跑了`

该场景对应的是用户的“止盈过早”和“离场纪律”问题。

## 8. 历史价格自动匹配

为了降低用户输入成本，并增强 Demo 的真实感，产品支持 A 股历史价格自动匹配。

用户选择 A 股和日期后，系统会自动匹配该日期的真实日 K 收盘价，并回填到价格输入框。如果所选日期不是交易日，例如周末或节假日，系统会自动匹配最近一个交易日的收盘价。

设计原因：

- 降低用户手动查历史价格的成本
- 提升计算结果的可信度
- 让复盘过程更接近真实交易数据
- 保留手动修改能力，兼容用户实际成交价不等于收盘价的情况

由于静态 HTML 对免费美股历史数据接口存在跨域、限流和授权限制，美股历史价格在 Demo 中保留手动输入。正式产品中可接入 Polygon、Finnhub、Twelve Data、Alpha Vantage 或券商行情 API。

## 9. 结果呈现设计

用户点击计算后，系统生成结果卡，包含：

- 股票名称
- 场景类型
- 关键日期
- 情绪化结果标签
- 金额差异
- 可购买力换算
- 复盘报告
- 分享图生成
- 复制文案

结果默认不提前展示，只有用户点击计算后才出现。这样设计是为了避免页面一开始就出现“预设答案”，让用户感知到这是基于自己输入生成的结果。

## 10. 可购买力换算

金额差异会被换算成更容易感知的物品，例如：

- 🚗 迈巴赫 S480
- 💻 RTX 5090
- 📱 iPhone 17 Pro Max
- 🏠 深圳别墅

直接展示金额是理性的，但用户对金额的体感并不稳定。将金额换算成具体物品，可以提升结果的感知强度和传播性。

这部分承担的是用户分享和情绪表达价值。

## 11. 复盘报告设计

「持仓平行宇宙」的核心不是制造后悔，而是帮助用户完成复盘。

系统会根据不同场景生成不同复盘方向：

- 想买没买后上涨：复盘“看对方向但没有执行”，引导用户思考为什么没有买入
- 想买没买后下跌：复盘“没有出手也是一种有效决策”，帮助用户识别当时的风险信号
- 卖出后继续上涨：复盘“止盈策略是否过于一次性”，引导用户思考底仓、分批止盈或移动止盈
- 卖出后下跌：复盘“离场纪律是否有效”，确认卖出是否基于止损、止盈或风险变化

这个设计将用户的情绪转化为数据，再将数据转化为下一次交易可以参考的规则。

## 12. 产品差异化

### 12.1 从交易执行延伸到交易复盘

大多数交易平台重点服务于行情查看和下单执行。本 Demo 增加了交易后的复盘能力，补足了用户在交易闭环中的最后一环。

### 12.2 从盈亏展示升级为决策解释

传统产品通常只展示“赚了多少/亏了多少”。「持仓平行宇宙」进一步解释：

- 为什么会少赚
- 为什么反而多赚
- 没买是不是一定错误
- 卖出是不是一定失败

它将结果展示升级为决策解释。

### 12.3 从工具属性延伸到传播属性

通过情绪标签、可购买力换算和分享图，功能具备一定传播性。

但传播不是目的，传播只是入口。最终仍然回到交易复盘和用户决策能力提升。

### 12.4 双端差异化体验

Web 端适合深度看盘和完整交易工作台，因此采用更专业、密度更高的布局。

Mobile 端适合快速查看、快速计算和快速分享，因此采用底部 Sheet 的轻量交互。

两个端保持同一套产品逻辑，但交互形态不同。

## 13. 产品闭环

「持仓平行宇宙」形成了一个完整的产品闭环：

1. 用户产生后悔或好奇
2. 选择股票和历史日期
3. 系统自动匹配历史价格
4. 用户点击计算
5. 系统生成结果卡
6. 用户获得金额差异和情绪反馈
7. 系统提供复盘报告
8. 用户可生成分享图
9. 用户沉淀下一次交易规则

这个闭环将“情绪入口 - 数据计算 - 行为复盘”连接起来。

## 14. 数据说明

本 Demo 使用真实股票案例，覆盖 A 股与美股。

当前版本的数据能力说明如下：

- A 股历史收盘价：支持自动匹配真实日 K 数据
- A 股非交易日：自动匹配最近交易日
- 美股历史价格：Demo 中保留手动输入
- 当前价格与交易数据：用于原型展示
- 下单、持仓、订单：本地模拟，不产生真实交易

正式产品中可进一步接入：

- 稳定行情 API
- 券商交易接口
- 用户真实交易记录
- AI 交易复盘模型
- 社区分享与复盘模板

## 15. 后续演进方向

### 15.1 AI 复盘助手

基于用户历史交易记录，自动识别追高、过早止盈、止损不及时、看对但不执行、仓位过重等常见问题，并生成个性化复盘建议。

### 15.2 策略对比

在“如果没卖”的基础上，增加更多策略模拟，例如一直持有、卖半留半、移动止盈、分批买入、定投持有，帮助用户看到不同策略下的收益和风险差异。

### 15.3 用户复盘档案

记录用户每次复盘结果，形成长期交易画像，例如用户最常错过的行业、最常卖飞的股票类型、风险承受区间、执行力问题和止盈止损习惯。

### 15.4 社区分享

将分享图作为轻量传播入口，形成交易复盘社区。用户可以分享“我卖飞了”“我躲过一劫”“我还没上车”“还好跑了”等结果，但产品需要避免鼓励盲目追涨，分享内容应始终引导回到复盘和风险意识。

## 16. 总结

本 Demo 的重点不是简单复刻一个股票交易页面，而是探索交易产品中一个容易被忽视的方向：

> 交易平台不应只帮助用户完成交易，也应帮助用户理解交易决策。

「持仓平行宇宙」通过历史数据、结果计算、情绪表达、可购买力换算和复盘报告，把用户的“如果当时……”转化为一个可使用、可解释、可传播的产品功能。

这是本 Demo 相较于普通交易界面的核心差异化。
